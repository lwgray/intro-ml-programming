## If you want to deepen your understanding

### Free online courses
- **Hugging Face Course** (free, comprehensive): https://huggingface.co/learn — most accessible next step
- **fast.ai Practical Deep Learning** (free): https://course.fast.ai — applied focus
- **Stanford CS231n (computer vision)**: free lectures + notes online
- **Stanford CS224n (NLP)**: free lectures + notes online
- **Andrej Karpathy's "Let's build GPT"** (YouTube): implement an LLM from scratch

### Books
- **"Deep Learning" by Goodfellow, Bengio, Courville**: comprehensive theory (free PDF online)
- **"Hands-On ML with Scikit-Learn, Keras, and TensorFlow"** by Aurélien Géron: practical
- **"Designing Machine Learning Systems"** by Chip Huyen: production ML

### Papers (start here)
- "Attention Is All You Need" (2017) — the transformer
- "GPT-3" (2020) — scale and emergent abilities
- "InstructGPT" (2022) — RLHF
- "Latent Diffusion Models" (2022) — Stable Diffusion
- "DPO: Direct Preference Optimization" (2023) — alternative to RLHF

## If you want to apply ML to YOUR work

### Hugging Face Hub
https://huggingface.co/models — thousands of pretrained models. Search by task. Most have ready-to-use code snippets.

### LangChain
https://python.langchain.com — orchestration framework for LLM apps. Chains, agents, RAG.

### LlamaIndex
https://www.llamaindex.ai — RAG (retrieval-augmented generation) framework.

### Streamlit / Gradio
For quick UI on top of ML models. Most ML-app demos use one of these.

### Modal / Replicate / Beam
Serverless GPU platforms for deploying models without managing infrastructure.

## If you want to build things (recommended for most)

The best way to learn ML beyond this course is to apply it to a real problem.

### Pick a project relevant to YOUR life
- Personal data analysis (your photos, emails, notes)
- Hobby project (your sport, hobby, music)
- Side business idea (automate a process, build a tool)

### Build POCs
1. Start with `pipeline()` for the easy path
2. Combine 2-3 models (like our Week 11 final)
3. Add a UI with Gradio or Streamlit
4. Deploy to a free tier somewhere

### Show your work
- GitHub for code
- Twitter/LinkedIn for what you built
- Hacker News / r/MachineLearning if it's interesting
- Talk about it at meetups

The compounding effect of building publicly is significant.

## Career paths (if you're considering)

### ML Engineer
Build ML systems. Strong programming + ML fundamentals + production skills. The course gave you a solid start.

### Applied scientist / Researcher
Develop new ML methods. Usually requires PhD or equivalent research experience.

### Data scientist / Analyst
Analysis-focused. Less ML, more statistics + business + communication.

### ML platform / DevOps
Build infrastructure for ML at scale. MLOps focus.

### Product / Strategy
Translate between ML and business. Less code, more strategy.

The course gave you broad exposure. Match to your strengths and interests.

## Honest closing note

You won't remember everything from this course. That's normal. What you SHOULD remember:

1. **The pattern: from problem → data → algorithm → evaluation**
2. **The architectures: trees for tabular, CNNs for images, transformers for sequences, generative for creation**
3. **The production-default: pretrained models + fine-tune for your task**
4. **The mindset: define a loss, optimize via gradient descent**

Everything else is searchable. The mental model is what stays.
