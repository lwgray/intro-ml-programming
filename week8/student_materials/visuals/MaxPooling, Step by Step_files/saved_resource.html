Window at rows 1–2, cols 1–2: the four scores are 0, 1, 2, 3 — only the strongest, 3, moves on. The other three are forgotten.

2 · Why throw away 3 of every 4 numbers?

It keeps the strongest evidence

Each number is a filter's “I found my pattern here!” score. The max asks: was the pattern found anywhere in this neighborhood? A 9 next to a 4 — the 9 is the answer; the 4 adds nothing new.

It halves the map

28×28 → 14×14 → 7×7. Fewer positions to compute on — and every later 3×3 conv now covers twice the original image, which is how receptive fields grow so fast.

It learns nothing

MaxPooling2D((2,2)) has 0 parameters. Like ReLU, it's a fixed rule, not a learned layer — check model.summary() and you'll see “Param # 0”.

Same pattern, shifted one pixel — the max doesn't budge. That's translation tolerance: the network stops caring exactly where in the neighborhood the feature sat.

MaxPooling is a summarizer, not a learner: in every 2×2 neighborhood it keeps only “the strongest thing my filter found here” — shrinking the map, growing every later layer's view, and shrugging off pixel-level jitter, all for zero parameters.
