Batch Normalization

As a network trains, every layer keeps changing what it hands to the next one. BatchNorm re-centers those hand-offs so no layer has to chase a moving target.

1 · The problem

The same layer sees a different world every few hundred steps

Watch the inputs arriving at one hidden layer as training runs. Upstream weights keep updating, so the distribution drifts — its center and spread wander. The layer keeps re-learning “what do my inputs look like now?” instead of its actual job.

center −0.9 · spread 0.6
→
Step 800
center +0.6 · spread 1.1
→

Dashed line: where the layer “expects” inputs to sit. Every panel is the same layer — only training time has passed.

2 · The fix

Re-center, re-scale — then let the network choose

For each batch, BatchNorm standardizes the activations (subtract the batch mean, divide by the batch spread), then applies two learned knobs — γ stretches, β shifts — so the network can undo the normalization anywhere it turns out to be unhelpful. Nothing is lost; stability is gained.

Raw activations
one channel, this batch — off-center, wide
→
− μ, ÷ σ
Standardized
mean 0, spread 1 — every batch, every time
→
× γ, + β
Network’s choice
γ, β are trained by backprop like any weight
x̂ = (x − μ_batch) / σ_batch  then  y = γ · x̂ + β  // γ, β: one pair per channel, learned

In a CNN, the statistics are per channel: channel 2’s μ and σ come from channel 2’s feature map across every image in the batch and every pixel (N × H × W values → one mean, one std).

A 32-channel layer learns 32 γs and 32 βs — just 64 extra parameters.

4 · The gotcha

It behaves differently at train time and test time

Training

Uses this batch’s own μ and σ. Every batch is a little different, so the statistics are slightly noisy — a side effect that acts as mild regularization.

Inference

Uses running averages of μ and σ collected during training — so a single image works fine. This is exactly why model.eval() in PyTorch and training=False in Keras matter.

BatchNorm keeps every layer’s inputs in a predictable range, so each layer can learn its job without chasing a moving target — which means higher learning rates, faster and more stable training, and a little regularization for free.
