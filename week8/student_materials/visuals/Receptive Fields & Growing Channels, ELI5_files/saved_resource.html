ELI5 · Convolutional networks

Receptive Fields, and Why Channels Grow as Maps Shrink

A filter never moves — what changes is how much of the original image each activation can “see.” That one idea explains why deep layers need more channels, and what GlobalAveragePooling actually hands to the classifier.

1 · Receptive field

How much of the original image one activation can see

The receptive field of an activation is the patch of the input image that could possibly influence its value. Every filter is still just 3×3 — but a deep filter slides over a map whose every cell already summarizes a big patch, so its effective view of the original image is enormous.

input · 28×28 px
→
layer-1 map · 28×28
Early layer: a keyhole One activation is computed from a 3×3 patch — 9 pixels.
All it can possibly say: “edge here?”, “corner?”, “light–dark boundary?” There aren’t many distinct 9-pixel stories, so ~32 filters cover them.
input · 28×28 px
→
deep map · 3×3
Deep layer: a picture window After a few conv + pool rounds, one cell of a 3×3 map is computed from ~24×24 of the original 28×28 — nearly the whole garment.
That patch can contain a collar and a sleeve and a zipper. Many simultaneous stories → 32 filters are no longer enough to tell them apart.
one 3×3 conv → sees 3×3
two stacked → 5×5
each 2×2 pool doubles every later conv’s step: +2 → +4 → +8 px per layer
2 · The trade

Shrink the “where,” grow the “what”

Pooling throws away spatial detail — you can’t keep 28×28 positions forever. The usual convention compensates by adding channels: each position describes a bigger patch, so it needs a longer list of numbers to say everything that’s in it. You’re trading location precision for vocabulary.

28×28 × 32
tiny view, small vocabulary
“edge? corner?”
→
14×14 × 64
medium view, bigger vocabulary
“curve? stripe? cuff?”
→
7×7 × 128
huge view, rich vocabulary
“sleeve? collar? heel? lace?”

Each sheet is one channel — one filter’s “is my pattern here?” map. Fewer positions per sheet, but many more sheets: the description per location gets richer exactly as the location’s view gets wider.

3 · The payoff

GlobalAveragePooling keeps only the channels

GAP collapses each channel’s map to its average — one number per channel. Position is deliberately discarded (“a sleeve is a sleeve wherever it is”); what survives is a C-length concept vector. That vector is the classifier’s entire view of the image — so the channel count at the end is the richness of the final description.

Dense + softmax reads the C-number vector and scores the 10 classes.
More channels at the end = more columns in this vector = a finer-grained description for the classifier to work with.
4 · One caveat for Week 8

Your Week 8 net keeps 32 → 32 on purpose

The doubling convention above (32 → 64 → 128, VGG-style) is what most architectures do — and why they do it is this page. Week 8’s CNN deliberately stays at 32 → 32 (~110K params) so the CNN-vs-MLP comparison is a fair, matched-parameter fight. Same intuition, intentionally traded away for pedagogy.

Early layers see keyholes, so a small vocabulary of filters suffices. Deep layers see the whole garment, so they need a big vocabulary — and since GAP hands the classifier exactly one number per channel, that final vocabulary is the image description the classifier gets.
